Overview

The Renderer uses Smarty Template Engine ‚Äî one of the most popular and mature template engines.

Smarty is a template engine for PHP. More specifically, it facilitates a manageable way to separate application logic and content from its presentation.
This is best described in a situation where the application programmer and the template designer play different roles.

Smarty Template Engine is used for rendering and cache management, it has neither override nor fall-back mechanism.

The Render engine (renderer) ensures compatibility with legacy themes of XOOPS2 and XOOPS2-JP.

Templates are built with HTML, CSS, JavaScript and Smarty tags
‚Äî Smarty is used with custom delimiters very similar to HTML markup language that make it easier to design and develop components.

The Renderer module allows you to manage files more easily, even when they‚Äôre not all stored in one central folder or directory. Although, some Themes have a template directory including most, if not all template files to meet an organization's needs for customization and personalization.

The Template Management allows you to enable a Template Set and check the list of templates that you can edit, copy, download or delete.

Banner Management

Allows you to manage banner ads and display ads in your website.

You can place a banner of any type, in any location of the page, and on any module.

You may show a banner to visitors a certain number of impressions.

An impression is a metric used to quantify the number of digital views of a client banner.

Smarty Tags

Smarty generates web content through the placement of special Smarty tags within a document.

The following tags can be added to Themes and Templates.

Smarty tags will be processed and substituted with other code.

Random Banner display


    <{$xoops_banner}>

Display Banner if not empty


    <{if $xoops_banner and $xoops_banner!=""}><{$xoops_banner}><{/if}>

Display a specific Banner Id (bid)


    <{if $xoops_banner && bid=2}><{$xoops_banner}><{/if}>

Client login


    <{$smarty.const.XOOPS_URL}>/banners.php

The .htaccess File

.htaccess files are plain-text configuration files used by the Apache HTTP web server. They allow users to set directory level options without requiring access to the httpd.conf file, including www rewrite, hotlink prevention, custom error pages, caching, and redirects.
As such it is required that your server uses Apache, and a web host that allows htaccess files (the most popular hosts do).

The Cool URI

The term used to describe a Uniform Resource Identifier (URI) that does not change. This enables resources that have been linked to, bookmarked, or referenced in other ways to remain valid links for many years to come.

A cool URI is considered to be a foundation requirement for the Semantic Web. As described by Tim Berners-Lee in ‚ÄúCool URIs don‚Äôt change‚Äù (1998), URIs should be designed with three things in mind: simplicity, stability and manageability.

Template Management

The list of Template Sets is displayed in Template Management.

The distribution package comes with a Default Template Set.

When modules are installed, their templates become part of this Default Template set.

The Default Template Set cannot be deleted nor modified !

To customize the templates in your site, first copy the default template set.

Once you have a new Template Set, you can activate it by clicking the "Select" button in the Control column.

Only then template files can be edited in the Template File Management.

Template File Management

The list of template files allow editing and download each file.

You can upload a Template Set in tar.gz format through the admin interface.

Theme Design

Additional Themes

Themes can be installed from Xupdate Store or from theme-developers' websites etc.

Uploading Themes

Distributed themes are mostly compressed. To use a theme, the uncompressed files must be uploaded into the directory public_html/themes/.

The Theme folder must include theme.html and manifesto.ini.php

Theme Selection

Theme Management allows you to simple click the "Select" button next to the theme that you want to use as the default theme or main theme of your site.

Any theme that is "checked" in the "Control" column, will be available to users in the block "Themes".

If you install the "Themes" block and grant users permission, they can then select their favorite theme and customize the look and feel of the website.

Uninstall a Theme

The render system manages themes without using the database.

So, to delete a theme, simply delete its directory from the "themes" directory.
