Pico Module

Pico can run multiple instances as a Content Management System (CMS), Document and Knowledge Management, Wrapper, Portfolio, Gallery and much more. One of the most significant advantages of using Pico is the simplified installation and updates, granular permissions and editing workflow.

  • Pico is a Trust module, D3 for directory free and duplicable, that can run multiple instances.
  • The module provides management control in the front-end for static content and publishing workflows.
  • The module features a simple and granular access permissions for groups or a specific user.
  • Among other options, custom 'head' with stylesheet and JavaScript by module, category or content.
  • Track the history of your document and see past revisions from the front-end
  • Customized Smarty Forms, override options for each category, forum integration to manage comments, and much more...

Permissions

Subcategories and contents inherit permissions from the nearest parent category, in the absence of individual permissions, all inherit from the Top-level category, by default.

Only administrators can create subcategories. However, you can assign users and roles to create and mange subcategories (either individually, or for all subcategories on your module instance). For example, you may grant users a specific role with the ability to create content.

On the front end, click "Edit category", then click "Category permissions"

On the back end, select the category and click "Submit" to manage users permissions.

Import and Synchronize

The import tool and data synchronization can both be used to populate an instance of Pico with data imported from other instances of modules Pico and TinyD.

The Import tool is intended primarily for bulk data population. Note that this resets the instance importing the data, which means losing all existing data and custom settings.

The data synchronization is intended to synchronize incremental changes on the instance deployed.

It’s important to use these tools correctly in order to ensure that your contents are imported and synchronized.

In addition, it is recommended to check and correctly set user permissions. This will prevent unintended access, helping to maintain data accuracy and ensure sensitive information is only seen by users with permissions.

Warning : All data in this module will be lost when you execute to import.

Import data

  1. Select a module : Pico or TinyD
  2. Click the action button import.

Synchronize redundant data

Data redundancy occurs when the same piece of data exists in multiple places, whereas data inconsistency is when the same data exists in different formats in multiple tables. Unfortunately, data redundancy can cause data inconsistency, which can result in contents with unreliable information.

Execute if your categories or contents display contradictory data.
