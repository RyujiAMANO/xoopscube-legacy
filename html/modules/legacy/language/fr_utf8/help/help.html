Overview

Les préférences système vous permettent de personnaliser les paramètres généraux de votre plate-forme, tels que le nom du site et le slogan, la langue et le thème, activer la recherche globale intégrée, fermer votre site pour maintenance, désigner un module pour votre page d'accueil et bien plus encore.

Panneau de configuration »» Système »» Préférences

Les paramètres de configuration sont agrégés et organisés en plusieurs modules principaux pour garantir la fiabilité, la disponibilité et la possibilité de faire évoluer votre application Web. Les modules de base sont nécessaires pour accomplir les activités d'entrée, de traitement, de sortie, de stockage, de mise à jour et de contrôle. L'accès aux principales fonctions, module utilitaire ou service, de configuration et de gestion sont représentés par les icônes du menu latéral du Panneau de configuration.

Pour en savoir plus, vous pouvez accéder à l'aide intégrée dans le sous-menu de chaque module et activer l'aide contextuelle (panneau d'administration).

Mise en route

Cette section fournit un guide de démarrage rapide avec un bref résumé du panneau de commande, du tableau de bord et de la liste de contrôle post-installation.

Guide de démarrage rapide

Une fois l'assistant d'installation terminé, votre site Web est préconfiguré avec les paramètres de configuration par défaut. Ces paramètres font référence à la valeur préexistante d'un paramètre configurable par l'utilisateur qui est attribué à votre plate-forme d'application Web. Par conséquent, vous devez effectuer certaines tâches de post-installation avant de lancer votre nouveau site Web. Celles-ci incluent la configuration du nom et du slogan du site, la personnalisation des balises méta (SEO), l'autorisation des visiteurs à s'inscrire et les autorisations des groupes d'utilisateurs. Des documents plus détaillés sont disponibles et seront mis en évidence tout au long de ce guide.

Stocker et gérer les préférences et les paramètres

Enregistrez vos préférences pour les blocs de tableau de bord, les astuces et les conseils. Gérez votre configuration et vos modules de manière efficace et fiable. Récupérez et déployez des modules complémentaires en temps réel à partir de X-Update Manager sans affecter les utilisateurs, en évitant les redéploiements chronophages. La plate-forme d'applications Web est basée sur la simplicité, la sécurité et la scalabilité.

Panneau de configuration et tableau de bord personnalisé

Le panneau de contrôle, également appelé zone d'administration, vous donne accès à toutes les actions pouvant être effectuées sur votre plate-forme d'applications Web. Il comprend également un tableau de bord qui peut être personnalisé depuis le "panneau d'administration". Vous pouvez activer ou désactiver les blocs d'administration, les conseils de module et les info-bulles d'action. Vos préférences sont stockées dans le localStorage de votre navigateur.

How to get help

There are a few different ways to find help for your Web Application Platform. You can use the Action Search to assist you in searching modules Help files and preferences. You can learn more about a module from the contextual Help dialog and the built-in User Guide.

Use Action Search

Click the x-cube on the right to open the side panel. Enter different keywords in the action search field to find what you are looking for in the modules User Guide files and modules Preferences.

Use Built-in Help

All modules installed on your Web Application Platform have a built-in User Guide. In the left side-menu, for each module, there is a help link, so you can easily get the User Guide for the modules you are using. A module's User Guide can provide easy access to other sections, topics, and examples.

Use Contextual Help Dialog

The Help button is available in the administration of most modules. This displays a modal dialog box in response to user action, to get the relative help section. For example, this button displays the contextual help for module managent.

Block management

The block management is part of the core system. Blocks are made available in block management by installing and activating modules but also when creating custom blocks. Simply put, there are two main types of blocks : Modules blocks and Custom blocks.

Modules blocks

Modules blocks are pre-built blocks such as "User Login", "Main Menu" or "Who's online". Each module block provides specific functionality and has its own settings and template.

Custom blocks

Custom blocks are a quick and easy way to extend and customize your content with images, text or third party services: maps, videos, etc. By default, Custom blocks are rendered with a dummy template. However, it is possible to create reusable components with their own custom template. We'll see more details about custom components later.

Block settings

  • Title
  • Side
  • Order
  • Cache time
  • Target modules
  • Target groups
  • Template

Block Side Position

The block side position is a legacy code convention to identify placeholders, also known as areas, regions or zones in the theme layout.

The theme specifies the overall page appearance, style and defines the placeholders available. Where a module block template can specify its own shape, size, and position.

>Along with more savvy combinations of custom templates and components, it's also important to understand how blocks are displayed and sorted (order/weight).

Block Placeholders

Your site's theme defines the placeholders available. Therefore, block placement in placeholders is done per theme on the Block Management. If you have more than one theme enabled on your site blocks might be placed differently for each theme.

Example of a common theme layout, structured as a 3 column portal with placeholders displayed as wireframes.

Defining placeholders in the theme layout is a minor design constraint. In fact, a framework is necessary in any creative process. Furthermore, given the modular architecture and the flexibility of the rendering engine, it is possible to create compositions adapted to the most diverse needs and thus deliver tailor-made solutions.

Block Order

The block management screen also lets you specify the vertical sort-order of the blocks within a theme placeholder. This can be done by the block numeric input field next to the block title. A spinner allows you to increment and decrement the value to the desired position. When editing the theme you can also customize a block by assigning a weight to the CSS style. Lighter blocks (smaller weight) are displayed up towards the top of the placeholder. Heavier blocks towards the bottom of it.

Block actions

After adding a block, click the edit icon in the block header to configure it.

Edit the new block - HTML

The Block Title includes a 'kebab' (three vertical dots) menu

The kebab menu, also known as the vertical three-dot menu, is an icon used to open a menu and display actions for the related item : quick links to block management.

Uninstall a Block

  • A Module block can be installed or uninstalled without being deleted from the system. While a Custom Block can be completely removed from the UI and the database.
  • If you uninstall a Module block, it is moved back to the management list of 'Install block'.
  • Uninstalled blocks keep their settings, so you can re-install the block and recover its last settings.

Module management

In a modular architecture, Each module is responsible for managing certain features and parameters. The required functions for configuring and managing The web application platforma are grouped into eight modules : components, system, render, user groups, profile, cache, message, protector.

Active modules are represented by the icons in the Side menu of the Control Panel.

Modules mostly work independently of one another. However, in a number of cases the functionality of certain modules depends on the capabilities of other modules. For example:

  • The module Pico can be used to manage articles but depends on D3Forum comment-integration to allow comments.

Word Censor

Control Panel > Preferences > Word Censorship

This section of the Control Panel allows you to enable word censoring. It is also where you define the list of censored words.

The profanity filter, sometimes referred to as just "filter" or "censor", is a feature typically used on forums and comments that automatically scans users' posts as they are submitted and automatically changes or censors particular words.

Enable censorship?

You may enable or disable word censoring. If you select “Yes”, the system will replace any specified words in channel entries, comments, forum posts, etc. according to your preference below.

Replacement characters

You may optionally specify a word or phrase to be used when replacing censored words. For example, if you set “#OOPS#” as your replacement word, and “shucks” is in your censored list, then anytime “shucks” is used it will be replaced with “#OOPS#”. If you do not set this preference, default "#OOPS#" will be used for each word that is censored.

Words to censor

Enter the censored words in the textarea using a common delimiter character "|" (pipe) to separate the values.

Set the characters replacing all the censored words in the input field "censor-replace".
