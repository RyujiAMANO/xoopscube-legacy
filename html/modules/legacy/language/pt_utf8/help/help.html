Introdução

Esta seção fornece um guia de início rápido com um breve resumo do painel de controle, painel e lista de verificação pós-instalação.

Guia rápido

Depois que o assistente de instalação for concluído, seu site será pré-configurado com as configurações padrão. Essas configurações referem-se ao valor pré-existente de uma configuração configurável pelo usuário que é atribuída à sua plataforma de aplicativo da web. Portanto, você precisa executar algumas tarefas de pós-instalação antes de lançar seu novo site. Isso inclui configurar o nome e o slogan do site, personalizar meta tags (SEO), permitir que os visitantes se registrem e permissões de grupos de usuários. Documentos mais detalhados estão disponíveis e serão destacados ao longo deste guia.

Salvar e gerenciar preferências e configurações

Salva as configurações de todos os blocos do painel. Gerencie configurações e módulos de forma eficaz e confiável. Recupere e implante complementos em tempo real do X-Update Manager sem afetar os usuários, evitando reimplantações demoradas. A plataforma de aplicativos da web é construída com simplicidade, segurança e escalabilidade.

Painel de controle e tema personalizado

O painel de controle, também conhecido como área de administração, dá acesso a todas as ações que podem ser executadas em sua plataforma de aplicativos da web. Também inclui um painel que pode ser personalizado a partir do "painel de administração". Você pode habilitar ou desabilitar blocos administrativos, dicas de módulo e dicas de ferramentas de ação. Suas preferências são armazenadas no localStorage do seu navegador.

Conteúdo principal

A ideia principal de ter um painel personalizável em XCL é fornecer acesso fácil a diferentes seções com um simples clique. Para conseguir isso, escolhemos os principais recursos do XCL que os administradores usam com mais frequência diariamente. Preenchemos o painel com "blocos administrativos" padrão que fornecerão algumas informações sobre o ambiente.

Você pode adicionar seus próprios widgets ao painel para diferentes finalidades - por exemplo, você pode incorporar qualquer API de terceiros ou widgets de monitoramento. Resumindo, os Widgets são uma solução rápida e não técnica, enquanto a API é um processo mais envolvente que requer conhecimento técnico específico.

Admin Theme Options

Show or hide blocks and modules Tips

Modules Tips are small alert messages that display short information about functionality.

Actions Tips display descriptive text when you rest the pointer on a button command or control.

Module Help enhanced Help are larger modals that display more descriptive text than a Module Tip and can have a link to a Help article.

You can set an option to show or hide Tips.

Show or hide Tips

  1. Click the x-cube admin panel
  2. Click theme options.
  3. Click help tips.
  4. Under User interface options, in the Admin Theme Options list, click the option that you want:

Show feature descriptions in Tips This option turns on Tips and Enhanced Help-Tips so that you see more information about a module feature, including the functionality name, keyboard shortcuts, and links to Help articles. This is the default setting.

Don't show feature descriptions in Tips This option turns off Enhanced Help Tips so that you see only the brief information and maybe a keyboard shortcut.

Don't show Tips This option turns off Tips and Enhanced Help Tips so that you only see the module's view and action controls.

Blocks management

The block management system is built into the kernel and provides five default block positions, the order of display, in which module, for which user groups and cache settings.

The block position is just a legacy code convention to identify areas in the theme layout.

The block template specifies the shape, and the global theme specifies the areas and appearance. However, it can be overridden by a custom template.

There are two types of blocks : Modules blocks and Custom blocks.

Modules blocks

Modules have several pre-built blocks that offer specific functionality and favor composition over inheritance. These blocks are made available in the block management by enabling modules.

Custom blocks

A Custom block can be built with HTML, CSS, JavaScript, PHP and Smarty variables. Thus allowing to create reusable Components to create an effective website layout.
It can also embed code or integrate with third-party features specific to a site's unique needs.

The block editor will switch according to the type of content.

The block management system is built into the kernel. It provides a text comment box where users can generate their own comments and leave them on a specific piece of content.

You can follow the last comments of your articles on your Dashboard. In addition to having a quick view, you can also give Reply to the comment as well as moderate its status as a public, unpublish, spam, and delete it.

Category management

Here you can create and edit categories, set the maximum file size, maximum image width and height, display and hide categories and images.

The following options are available when creating a category:

Name

A descriptive name of the category.

Permissions

  • Groups who can upload to the category.
  • Groups who can view the category.

File size

You can optionally set a maximum allowed size in kilobytes for uploaded file. Leave blank if you do not wish to impose a limit.

Example: 1024000 bytes = 1,000Kb = 1MB

Note: Hosting services usually have limits on the amount of data that can be uploaded via PHP If you need to upload very large files, you should contact your server's support service and tell them about the size limits they have put in place. Alternatively, you can install the module Xelfinder, the web file manager can connect to a cloud service.

Image width

Optionally set the maximum allowed width in pixes for uploaded images. Images over this setting will be automatically resized.

Image height

Optionally set the maximum allowed height in pixes for uploaded images. Images over this setting will be automatically resized

Display

Show or hide the category

Order

In the list component, you can sort by any column. Leave the value by default for the original sort order, or set a custom order with a numeric value.

Category store type

Choose either the file or the database storage scheme for your image files.

This parameter cannot be changed later!

Keep in mind that if you select the database as the storage destination, image files will be stored as BLOB type data which may not be suitable for backup purposes.

Word Censor

Control Panel > Preferences > Word Censorship

This section of the Control Panel allows you to enable word censoring. It is also where you define the list of censored words.

Enable censorship?

You may enable or disable word censoring. If you select “Yes”, the system will replace any specified words in channel entries, comments, forum posts, etc. according to your preference below.

Replacement characters

You may optionally specify a word or phrase to be used when replacing censored words. For example, if you set “tisk tisk” as your replacement word, and “shucks” is in your censored list, then anytime “shucks” is used it will be replaced with “tisk tisk”. If you do not set this preference, a pound symbol will be used for each character that is censored, so “shucks” would be converted to “######“.

Words to censor

Specify a list of words to censor. Wildcards are allowed. For example, test* would censor the words “test”, “testing”, and “tester”, while *gress would censor the words “progress” and “congress”.

The profanity filter, sometimes referred to as just "filter" or "censor", is a feature typically used on forums and comments that automatically scans users' posts as they are submitted and automatically changes or censors particular words.

Enter the censored words in the textarea using a common delimiter character "|" (pipe) to separate the values.

Set the characters replacing all the censored words in the input field "censor-replace".

Check the GitHub repository of Shutterstock list of bad words used for filtering
